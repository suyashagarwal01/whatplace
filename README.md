# WhatPlace – Chrome Extension

One-click identification of the **location** shown in the current image or video (YouTube, Vimeo, any site). No extra steps: you click the button and the extension captures the screen, sends it to AI vision, and shows you the place.

## How it works

1. You open a page with an image or video (e.g. a YouTube vlog with a city in the background).
2. You click the extension icon and then **Find Location**.
3. The extension captures the visible tab, sends it to **Google Gemini** (vision), and displays the identified location in the popup.

All processing is done by Google’s API; the extension only captures the screenshot and shows the result.

## Setup

### 1. Get a free Gemini API key

- Go to [Google AI Studio](https://aistudio.google.com/apikey).
- Sign in and create an API key (free tier is enough).

### 2. Install the extension

1. Open Chrome and go to `chrome://extensions/`.
2. Turn on **Developer mode** (top right).
3. Click **Load unpacked**.
4. Select the `WhatPlace` folder (the one that contains `manifest.json`).

### 3. Set your API key

1. Click the WhatPlace extension icon.
2. Click **Settings (API key)** at the bottom.
3. Paste your Gemini API key and click **Save**.

## Usage

1. Open any page with an image or video (e.g. YouTube, Vimeo, Instagram, or any site).
2. Make sure the **location you care about is visible** on the screen (e.g. the video is playing and the right frame is shown).
3. Click the WhatPlace icon, then **Find Location**.
4. Wait a few seconds; the identified location will appear in the popup.

No need to right‑click, save images, or leave the page—one click is enough.

## Why Gemini?

- **Free tier**: Gemini 1.5 Flash has a generous free quota (e.g. 15 RPM, 1500 requests/day).
- **Vision**: It can analyze images (your screenshot) and describe places accurately.
- **No backend**: The extension talks to Google’s API directly; you don’t need a server.

You can replace the API key with another provider later if you add support for it in the code.

## Permissions

- **activeTab** – To capture the visible tab when you click the button.
- **storage** – To store your API key locally (sync across your Chrome profile).
- **host_permissions** – To call the Gemini API from the extension.

Screenshots are sent only to Google’s API for analysis; they are not stored by the extension.

## Troubleshooting

- **“API key not set”** – Open Settings from the popup and save your Gemini API key.
- **“Invalid API key or access denied”** – Check the key at [Google AI Studio](https://aistudio.google.com/apikey) and ensure the Gemini API is enabled.
- **“Rate limit exceeded”** – The free tier allows about 15 requests per minute. The extension will disable the button for 60 seconds after a rate limit so you can wait and try again without hitting the limit repeatedly.
- **404 / “model not found”** – The extension now tries several Gemini models in order (2.5 Flash, 2.0 Flash, 1.5 Flash, 1.5 Pro on both v1 and v1beta). If you still see 404, your API key or region may have different models; create a key at [Google AI Studio](https://aistudio.google.com/apikey) and ensure you’re in a supported region.
- **Wrong or no location** – Try pausing the video on a clear shot of the place and run Find Location again.
